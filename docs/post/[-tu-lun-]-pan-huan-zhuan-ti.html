[图论]判环专题

在写图论问题的时候经常会遇到一些判环问题，本人也没有细致总结过，想根据自身能力总结一下，当然后续学会了更多的技巧会把这篇博客完善。

有向图判环

有向图判环：五大核心算法横向评测

1. 拓扑排序 (Kahn 算法)

原理介绍：
基于贪心与 BFS 框架。每次剥离图中入度为 0 的节点，并删除该节点连出的所有有向边。如果图中存在环，那么环上所有节点的入度永远不可能减为 0，从而导致这些节点永远无法入队。最后通过对比进入过队列的节点总数与原图总节点数 V 是否相等，即可判定是否有环。

适用范围：

  • 纯粹的 DAG（有向无环图）判定。
  • 需要求解依赖关系、调度问题（如课程安排、任务执行次序）。
  • 结合优先队列求字典序最大/最小的拓扑序列。

时间复杂度：O(V + E)

难用的地方（痛点）：

  • 无法提取路径：拓扑排序只能告诉你“图里有环”，但如果你需要输出环的具体节点序列，Kahn 算法几乎无能为力。
  • 信息丢失：在剥离入度为 0 的点的过程中，图的原始结构被破坏了，不方便做后续的图上回溯操作。
    代码实现：
int n, m;
int h[N], e[M], ne[M], idx;
int deg[N];
int q[N], hh = 1, tt = 0;
void add(int a, int b)
{
    e[idx] = b, ne[idx] = h[a], h[a] = idx++;
}
bool topsort()
{
    for (int i = 1; i <= n; i++)
    {
        if (deg[i] == 0)
        {
            q[++tt] = i;
        }
    }
    while (hh <= tt)
    {
        auto u = q[hh++];
        for (int i = h[u]; ~i; i = ne[i])
        {
            int j = e[i];
            if (--deg[j] == 0)
            {
                q[++tt] = j;
            }
        }
    }
    return tt == n;
}

void init()
{
    idx = 0;
    for (int i = 1; i <= n; i++)
    {
        h[i] = -1;
    }
}

2. DFS 三色标记法

原理介绍：
通过深度优先搜索遍历图，并为每个节点维护三种状态（颜色）：

  • 0（白色）：完全未被访问过。
  • 1（灰色）：正在当前的递归调用栈中。
  • 2（黑色）：该节点及其所有子节点都已完全搜索完毕，出栈。

在 DFS 过程中，如果尝试访问一个相邻节点且发现它的状态是 1，说明找到了一条指向祖先节点的“后向边（Back Edge）”，即发现了一个环。

适用范围：

  • 快速判环，代码极简。
  • 必须找出环的具体路径的最优解（通过递归栈或记录前驱节点很容易还原路径）。

时间复杂度：O(V + E)

难用的地方（痛点）：

  • 爆栈风险：由于是递归实现，如果图是一条极长的链（如 10^5 级别），在某些不支持尾递归优化或栈空间严格限制的评测机上可能会引发 Stack Overflow（尽管现代比赛平台对 C++ 栈空间放得很宽，但仍需警惕）。
  • 多连通块：不能只从一个点开始 DFS，必须在外层套一个循环，确保所有状态为 0 的孤立点或独立分量都被遍历到。
    代码实现：
int n, m;
int h[N], e[M], ne[M], idx;
int col[N]; // 0:白色(未访问), 1:灰色(递归栈中), 2:黑色(访问完毕)
int stk[N], top=0;     
vector<int> cycle;  
void add(int a, int b)
{
    e[idx] = b, ne[idx] = h[a], h[a] = idx++;
}
bool dfs(int u)
{
    col[u] = 1;        
    stk[++top] = u;  

    for (int i = h[u]; ~i; i = ne[i])
    {
        int j = e[i];
        if (col[j] == 1) 
        {
            int p = top;
            while (p > 0 && stk[p] != j) p--; 
            
            for (int k = p; k <= top; k++) cycle.push_back(stk[k]);
            return true; 
        }
        else if (col[j] == 0) 
        {
            if (dfs(j)) return true; 
        }
    }

    col[u] = 2; 
    top--;  
    return false;
}

bool ck()
{
    for (int i = 1; i <= n; i++)
    {
        if (col[i] == 0) 
        {
            if (dfs(i)) return true;
        }
    }
    return false;
}

void init()
{
    idx = top = 0;
    cycle.clear();
    for (int i = 1; i <= n; i++) h[i] = -1, col[i] = 0;
}

3. Tarjan 强连通分量算法

原理介绍：
同样基于 DFS，但引入了时间戳 dfn 和追溯值 low。它将图划分为若干个“强连通分量（SCC）”。如果一个图中存在大小大于 1 的 SCC，或者存在一个大小为 1 但带有自环的 SCC，则说明图中包含环。

适用范围：

  • 图论进阶操作的基石：缩点。
  • 需要将有环图重构为 DAG 以便进行动态规划（DP）或拓扑排序。
  • 统计环内部的点权/边权信息。

时间复杂度：O(V + E)

难用的地方（痛点）：

  • 代码繁琐：相比于前两者，Tarjan 的模板更长，涉及栈的维护以及 dfn 与 low 的交叉更新。
  • 极易写错的细节：在更新 low 值时，区分树边和横叉边/后向边非常关键（例如 if (in_stack[v]) low[u] = min(low[u], dfn[v]); 这里如果不小心写成了 low[v] 可能会在复杂图中出错）。
  • 杀鸡用牛刀：如果题目仅仅是问 Yes/No，写 Tarjan 会增加不必要的 debug 成本和常数开销。

4. SPFA / Bellman-Ford 算法 (负环特化)

原理介绍：
基于最短路中的“松弛操作”。在一个有 V 个节点的图中，任意两点之间的最短路径最多包含 V-1 条边。如果存在某条最短路径包含 >= V 条边，或者某个节点被放入 SPFA 队列的次数 >= V，则说明图中存在可以通过不断绕圈来无限减小路径长度的“负权环”。

适用范围：

  • 带权有向图的负环判定。
  • 差分约束系统的前置判解。
  • 01 分数规划（二分平均值）的检验条件。

时间复杂度：最坏 O(V * E)

难用的地方（痛点）：

  • “关于 SPFA，它死了”：极其容易被出题人的特殊构造数据（如网格图、菊花图）卡成最坏的指数级或 O(V * E) 复杂度，导致 TLE。
  • 多测不清空，爆零两行泪：队列、入队次数数组 cnt、最短路数组 dis、访问标记 vis 在多组数据测试下极其容易漏清空。
  • 起点问题：图可能不连通，从单一节点出发可能无法探测到其他连通块中的负环，通常需要建立一个虚拟超级源点连接所有节点。
    代码实现：
int n, m;
int e[N], ne[N], h[N], idx, w[N];
bool st[N];  // 标记节点是否在队列中
int cnt[N];  // 记录每个节点的松弛次数
int dist[N]; // 记录从起点到各点的最短距离
queue<int> q;
void add(int a, int b, int c)
{
    e[idx] = b;
    w[idx] = c;
    ne[idx] = h[a];
    h[a] = idx++;
}
bool spfa()
{
    memset(dist, 0x3f, sizeof dist);
    for (int i = 1; i <= n; i++)
    {
        q.push(i);
        st[i] = true;
    }
    while (q.size())
    {
        auto t = q.front();
        q.pop();
        st[t] = false;
        for (int i = h[t]; i != -1; i = ne[i])
        {
            int j = e[i];
            if (dist[j] > dist[t] + w[i])
            {
                dist[j] = dist[t] + w[i];
                cnt[j] = cnt[t] + 1;
                if (cnt[j] >= n)
                    return true;
                if (!st[j])
                {
                    st[j] = true;
                    q.push(j);
                }
            }
        }
    }
    return false;
}

void init()
{
    idx = 0;
    for (int i = 1; i <= n; i++)
    {
        h[i] = -1;
    }
}

5. 快慢指针 (Floyd 判环法)

原理介绍：
设置两个指针（通常代表状态或节点下标），slow 每次前进一步，fast 每次前进两步。如果图中存在环，fast 必然会在环内追上并等于 slow。追上后，将其中一个指针拨回起点，两者同速前进，再次相遇的点即为入环的起点。

适用范围：

  • 每个节点出度严格为 1 的有向图（内向森林 / 基环树 / Functional Graph）。
  • 状态机转换、置换群问题。
  • 内存空间极度受限（如 O(1) 空间要求）或无法建图的黑盒交互题。

时间复杂度：O(V)

难用的地方（痛点）：

  • 极度受限的图结构：只要图中有一个点的出度大于 1（即存在分支），这个方法就彻底瘫痪。
  • 只能找环，无法顺带处理图的其他拓扑性质。
    代码实现：
int ne[N];

int floyd_find_cycle(int head)
{
    int l = head;
    int r = head;

    while (r != -1 && ne[r] != -1) 
    {
        l = ne[l];
        r = ne[ne[r]];

        if (l == r)          
        {
            l = head;           
            while (l != r)   
            {
                l = ne[l];
                r = ne[r];
            }
            return l;           
        }
    }

    return -1; 
}

无向图判环：四大核心算法横向评测

相比于有向图，无向图的边是双向的（即 A-B 等价于 A->B 且 B->A）。这就带来了一个致命的陷阱：在遍历时，极容易把刚走过来的路当作环（即 A 走到 B，B 又看了一眼 A）。因此，无向图判环的核心就在于——如何优雅地屏蔽“回头路”。

1. 并查集 (DSU) —— 无向图判环的绝对主力

原理介绍：
并查集天生就是为无向图连通性而生的。初始时，每个节点各自为一个独立的集合。接下来遍历图中的每一条无向边 (u, v)：

  • 如果 u 和 v 当前不在同一个集合中，说明这条边连接了两个不同的连通块，我们将它们合并。
  • 如果 u 和 v 已经在同一个集合中，这就意味着：在不通过当前这条边 (u, v) 的情况下，u 和 v 之间就已经有别的路相连了。现在再加上这条边，必定构成闭环！

适用范围：

  • 无向图判环的首选方案，思维难度极低。
  • 动态加边判环（在线判环）：边是一条一条给出的，问加到第几条边时出现了环。
  • Kruskal 最小生成树算法的底层核心。

时间复杂度：O(V + E * α(V)) （结合路径压缩，近乎 O(V + E)）

难用的地方（痛点）：

  • 彻底丢失路径信息：并查集只能极其高效地告诉你“有环”，但如果你问它“环是哪几个点”，它一问三不知。
  • 无法用于有向图。

代码实现：

int n, m;
int p[N]; 
int find(int x)
{
    if (p[x] != x) p[x] = find(p[x]);
    return p[x];
}

bool check_cycle()
{
    for (int i = 1; i <= n; i++) p[i] = i;
    for (int i = 0; i < m; i++)
    {
        int u, v; 
        // cin >> u >> v;
        int pa = find(u), pb = find(v);
        if (pa == pb) 
        {
            return true; 
        }
        else 
        {
            p[pa] = pb; 
        }
    }
    return false;
}

2. DFS 遍历 (传父节点法)

原理介绍：
无向图同样可以用 DFS 判环。由于一条无向边会在邻接表中存两次（u->v 和 v->u），为了避免发生 u 搜到 v，v 又立刻顺着反向边搜回 u 的误判，我们需要在 DFS 递归时，额外传递一个参数 fa（父节点/上一步的节点）。
如果在 DFS 过程中，访问到了一个已经被标记访问过的节点，且这个节点不是当前的父节点 fa，则说明找到了一条真正的“后向边”，图中有环。

适用范围：

  • 必须找出无向环的具体路径。
  • 图论基础搜索操作。

时间复杂度：O(V + E)

难用的地方（痛点）：

  • 重边陷阱：如果两个节点之间有两条一模一样的无向边，传 fa 的方法会失效（因为它仅仅屏蔽了节点编号）。严谨的做法是传“上一条走过的边的编号（edge_index）”来避免原路返回。
  • 依然存在深层图的爆栈风险。

代码实现：

int n,m;
int h[N], e[M], ne[M], idx;
bool vis[N];
int pre[N];        
vector<int> cycle;  // 存储最终提取出来的环路径

void add(int a, int b)
{
    e[idx] = b, ne[idx] = h[a], h[a] = idx++;
}

bool dfs(int u, int fa)
{
    vis[u] = true;

    for (int i = h[u]; ~i; i = ne[i])
    {
        int j = e[i];
        if (!vis[j])
        {
            pre[j] = u; 
            if (dfs(j, u)) return true;
        }
        else if (j != fa) 
        {
            int now = u;
            while (now != j)
            {
                cycle.push_back(now);
                now = pre[now];
            }
            cycle.push_back(j); // 最后把起点 j 补进去
            return true;
        }
    }
    return false;
}

bool ck()
{
    cycle.clear();
    for (int i = 1; i <= n; i++)
    {
        if (!vis[i])
        {
            if (dfs(i, -1)) return true; 
        }
    }
    return false;
}

3. BFS 遍历 (层序剥离)

原理介绍：
BFS 判环的思想与 DFS 极其类似。我们将节点推入队列的同时，一并推入它的父节点。如果在向外扩展时，撞到了一个已经被访问过的节点，且该节点不是导致当前节点入队的父节点，则说明有环。此外，由于无向图可能存在多个不连通的连通块，必须在外层包裹一层全局扫描函数，确保没有遗漏。

适用范围：

  • 需要求最短环的长度（在无权图中，BFS 第一次发现环的时候，通常对应着一条较短的闭环）。
  • 避免递归导致的爆栈风险时的最佳替代方案。

时间复杂度：O(V + E)

难用的地方（痛点）：

  • 提取路径比 DFS 繁琐得多，需要额外倒推 pre 数组才能把环的两端拼接起来。
  • 代码略显臃肿，队列需要同时维护当前节点和父节点信息。

代码实现：

int h[N], e[M], ne[M], idx;
bool vis[N];
int q[N], pa[N];     // q 存当前点，pa 存父节点
int hh = 1, tt = 0;
int pre[N];          // 记录每个节点的前驱，用于提取路径
vector<int> cycle;   // 存储最终提取出来的环路径

void add(int a, int b)
{
    e[idx] = b, ne[idx] = h[a], h[a] = idx++;
}

// 针对单个连通块的 BFS 判环
bool bfs(int u)
{
    hh = 1, tt = 0;
    q[++tt] = u;
    pa[tt] = -1;
    vis[u] = true;

    while (hh <= tt)
    {
        int t = q[hh];
        int fa = pa[hh++];

        for (int i = h[t]; ~i; i = ne[i])
        {
            int j = e[i];
            if (!vis[j])
            {
                vis[j] = true;
                pre[j] = t; // 记录前驱
                q[++tt] = j;
                pa[tt] = t;
            }
            else if (j != fa)
            {
                // 撞见已被访问且非父亲的节点，确认有环！
                // 提取路径：从当前点 t 和冲突点 j 分别往回倒推
                cycle.push_back(j);
                int now = t;
                while (true)
                {
                    cycle.push_back(now);
                    if (now == u) break; 
                    now = pre[now];
                }
                return true; 
            }
        }
    }
    return false;
}

// 外层全局扫描，防止因图不连通而漏掉其他连通块里的环
bool ck_bfs()
{
    cycle.clear();
    for (int i = 1; i <= n; i++)
    {
        if (!vis[i])
        {
            if (bfs(i)) return true;
        }
    }
    return false;
}

4. 拓扑排序 (度数裁剪法)

原理介绍：
虽然拓扑排序主要针对有向图，但无向图也能借用这个思想。
在一棵没有环的无向树中，叶子节点的度数一定为 1（孤立点为 0）。如果有环，环上的节点度数一定 >= 2。
我们只需每次把图中度数 <= 1 的节点放入队列并删除，同时把它相连的节点的度数减 1。如果最后图中还有剩余的节点（即队列未能处理完所有点），那么这些剥不掉的“硬骨头”构成的必然是环！

适用范围：

  • 无向基环树（Pseudotree） 的绝杀算法：不仅能判环，还能极其干净地把环上的所有节点分离出来（最后度数 > 1 的点全都在核心环上）。

时间复杂度：O(V + E)

难用的地方（痛点）：

  • 图的结构被破坏。
  • 仅仅为了判个环用它，不如并查集来得轻量快捷。

代码实现：

int h[N], e[M], ne[M], idx;
int deg[N]; // 记录无向图中每个点的度数
int q[N], hh = 1, tt = 0;

bool check_cycle()
{
    hh = 1, tt = 0;
    for (int i = 1; i <= n; i++)
    {
        if (deg[i] <= 1) 
        {
            q[++tt] = i;
        }
    }
    while (hh <= tt)
    {
        int u = q[hh++];
        for (int i = h[u]; ~i; i = ne[i])
        {
            int j = e[i];
            if (--deg[j] <= 1)
            {
                q[++tt] = j;
            }
        }
    }
    
    return tt < n; 
}
